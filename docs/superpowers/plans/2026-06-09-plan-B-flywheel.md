# M9 Plan B — Data Flywheel → net2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pull the first turn of the self-play flywheel — regenerate ~100M training positions with the 0.8.0 NNUE engine (better play + labels than the HCE engine net1 learned from), retrain the **identical** `(768→768)x2→1` net → net2, and ship it only if it beats net1 by SPRT.

**Architecture:** Mostly **ops + measurement with verification gates**, plus one tiny code change (swap the embedded net file). The single isolated variable is *the strength of the engine that generated the training data* — datagen params, data volume, and the training recipe are all held identical to net1. net2 has the same shape, so the runtime is unchanged; net2.bin just replaces net1.bin.

**Tech Stack:** the `datagen` bin (0.8.0 build), `tools/trainer` (bullet CUDA, RTX 5080 sm_120), `tools/trainer/prepare-data.sh`, `tools/sprt.sh`, `tools/baseline.sh`, `tools/anchored-gauntlet.sh` (extended pool from Plan A).

**Sequencing & the idle-system rule:** datagen (~13h) and every gauntlet/SPRT each **saturate the CPU**, so they must run **serially, never concurrently** (one heavy job at a time on an idle system). The only safe parallelism with Plan A is Plan A's *light* tasks (acquiring binaries, editing scripts) — not its re-baseline gauntlet. net2's **anchored gauntlet (Task 8) is gated on Plan A** (it needs the extended pool); the SPRT (Task 7) is not.

**Key facts (verified):**
- **datagen runs from the current build's engine.** The 0.8.0 build already *is* the NNUE engine, so "regenerate with the 0.8.0 engine" = run `datagen` on the clean `main` tree (commit `72c124e`). Tasks 1–5 must run **before** the net2 swap (Task 6) so the data is generated by net1/0.8.0, not a half-built net2.
- datagen CLI (`src/bin/datagen.rs`): `cargo run --release --bin datagen -- [--out DIR] [--games N] [--threads T] [--seed S] [--nodes SOFT] [--opening-plies P] [--max-plies M] [--resign-cp CP] [--resign-plies N] [--tb PATH]`. Defaults: out=`tools/data/selfplay`, games=100000, threads=22, seed=1, nodes=5000, opening-plies=8, max-plies=400, resign-cp=1000, resign-plies=8. Shard line = `FEN | cp_white | wdl_white` (wdl ∈ {1.0,0.5,0.0}). Stats mode: `cargo run --release --bin datagen -- stats <DIR>` → prints `positions`, white W/D/L %, cp min/max/mean, and `LEAKS -> in-check: X bad-fen: Y (both MUST be 0)`.
- **net1's params:** seed 1, threads 22, nodes 5000, default openings/resign; output `tools/data/selfplay-net1/` (22 shards). net2 reuses **identical** params (same seed 1 → identical opening sequence → isolates the engine difference).
- trainer recipe (`tools/trainer/src/main.rs`): HIDDEN=768, QA=255, QB=64, SCALE=400, batch_size=16384, **superbatches=25**, **bps=6104** (= one ~100M-position epoch per superbatch), `wdl::ConstantWDL{value:0.2}`, `lr::StepLR{start:0.001, gamma:0.3, step:(superbatches*2/3).max(1)=16}`, save_rate=25, threads=4, output_directory=`checkpoints`. Net → `tools/trainer/checkpoints/<id>-<N>/quantised.bin`, **1,184,320 bytes** for H=768.
- prepare: `tools/trainer/prepare-data.sh <shard-dir> <out.bin> [mem_mb]` (cats shards → `bullet-utils convert --from text` → `bullet-utils shuffle`). net1 used `... tools/data/selfplay-net1 tools/data/net1.shuf.bin 24576`.
- SPRT: `tools/sprt.sh <new> <old> [elo1=10]` → fastchess tc=8+0.08, Hash16, Threads1, 8moves book, `-sprt elo0=0 elo1=10 alpha=0.05 beta=0.05`; prints `H1 accepted` / `H0 accepted`.
- baseline preserve: `tools/baseline.sh <name>` copies `target/release/nebchess` → `tools/bin/baseline-<name>` (gitignored). **The current build = net1, so this captures the SPRT baseline.**
- embed wiring: `src/eval/nnue/mod.rs` `embedded()` = `NnueEvaluator::from_bytes(include_bytes!("net1.bin"))`; `src/eval/nnue/net.rs` asserts size 1_184_320 (compile-time + runtime). `.gitignore` ignores `*.bin` with exception `!/src/eval/nnue/net1.bin`.
- 0.8.0 bench = 60525. **net2 will change the bench** (different weights → different search) — the wiring commit carries the new `Bench:`.

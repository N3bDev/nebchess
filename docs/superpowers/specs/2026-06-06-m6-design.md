## M6.1 — Search polish (the elo)

Gate order is dependency-fixed; canary before every SPRT; one feature per gate.

1. **SEE (new infrastructure) + qsearch SEE pruning.** Nothing in the engine computes static exchange today (ordering is pinned MVV-LVA). Implement the swap algorithm (tests against known exchange positions, incl. pins/x-rays at the documented approximation level), then skip losing captures in qsearch. Gate [0,5].
2. **Continuation history** (1-ply + 2-ply conthist tables, used in move ordering). Gate [0,10] (largest expected remaining search gain).
3. **History-driven LMR** (re-tune the reduction ladder; adjust reductions by history/conthist score). Gate [0,5].
4. **Singular extensions** (the `excluded_move` field has waited since M3; TT-move singularity verification search) **+ check extensions** (new — none exist today; "polish" in the original sketch was a misnomer). One gate [0,5].
5. **Futility v2 — the M4 IOU:** the futility-incident fix (c41d9c6) promised "deeper futility returns with gives_check/SEE margins at M6." Extend futility to d≤4 with gives_check and SEE guards. Gate [0,5]; canary watched closely (this exact feature fired the canary once).
6. **Margin sweep LAST:** futility/RFP/null/aspiration margins revalidated against the M5 eval scale — after all search changes land, because margins interact with everything. Probe-rank candidate margin sets, SPRT the winner [0,5]. (Context: hybrid Q mg = 1047 landed within 2cp of the M4 value the margins were tuned against, so this is tune-up, not emergency.)

## v0.5.0 Lichess field telemetry (user-reported 2026-06-07; shapes Plan 7 priorities)

Lab strength confirmed live (wins/draws vs 2400-2500 bots; converted a 2530 bot's mistake into forced mate; CCI-4 mate-in-10 executed once concrete). Four leak patterns, with a unifying hypothesis — **clock collapse is the probable root of three**:

1. **Clock collapse (the dominant leak):** strong/playable/winning positions, then 20-50 moves played on seconds. TimeBrain is the single highest-leverage practical item — priority confirmed by field data, not just plan reasoning.
2. **Perpetual-check leakage (main draw pattern):** better/pressing positions drawn via queen-check cages/repetition. Substantially a time-trouble artifact (cages form in the blitzed phase; repetition already scores draw_score=0, so the engine avoids it WHEN it sees it — depth, not knowledge, is the gap). Plan 7 work item: classify the draw games (forced perpetual vs leaked win) from the PGNs BEFORE designing any fix; only add search/eval mechanisms (e.g. escape-square awareness, contempt-adjacent draw avoidance) if leaks persist after TimeBrain.
3. **Endgame conversion immature:** R-vs-bare-K drawn (elementary at any sane depth — near-certainly clock collapse), pawn-race awareness late. Syzygy 3-4-5 priority confirmed; the KR-K-on-seconds case becomes a TimeBrain stress test (winning eval + low clock must still convert).
4. **Sacrifice entrances missed (Greek gift):** consistent with the T7 finding — unconditional check extensions lose elo, but ungated NOTHING extends the entrance. The M7+ candidate is GATED check extensions (SEE≥0 checks only / depth-capped). Plan 7 adds a small "sacrifice entrance" EPD suite built from the live-game misses as canary enrichment (suite-building only; the fix itself stays M7+).

## M6.2 — TimeBrain v1 (Plan 7)

Two gates, not six micro-gates (TM features interact; isolating them individually measures noise):
- **Gate 1 — allocation core:** soft/hard limit split; stable-PV early stop (best-move stability across iterations scales the soft limit down); emergency reserve (never allocate into the final buffer; flag protection).
- **Gate 2 — adaptivity:** eval-swing panic extension (the M4-deferred PV-instability item: score drops between iterations extend time); clock-ratio catch-up (behind schedule → speed up); won-position fast play (stable winning score → reduce).
- Move-time telemetry (`info string` allocated/used per move) rides along ungated.

**TM-specific gate protocol:** each gate = SPRT (8+0.08) + idle forfeit gauntlet (TM regressions are forfeits, not elo). Phase capstone: sudden-death stress (60+0 — Lichess has increment-free pools) + LTC probe (60+0.6), both zero-forfeit requirements.

## M6.3 — Bot practicals + pondering capstone (Plan 7)

- **Polyglot opening book:** engine-side reader (UCI `BookFile`/`BookDepth` options, weighted move selection, out-of-book detection) + builder tool (PGN → .bin; filters: both-players elo floor, minimum plies, junk/bot-game exclusion; result-weighted). Sources: `db/export_ELO2400.pgn` (748MB OTB ≥2400) = soundness base; `db/LumbrasGigaBase_Online_2025.pgn` (766MB Lichess Elite) = blitz-practicality weighting after heavy filtering. NOT tuning data (the big3 lesson). Gate [0,5] with match-openings off (book vs no-book).
- **Syzygy tablebases** via pyrrhic-rs: 3-4-5 men (~1GB, laptop-realistic; 6-men deferred to desktop). WDL probing at interior nodes (depth/ply-gated), DTZ at root; `SyzygyPath` UCI option; correctness suite on known TB positions. Gate [0,5].
- **Lichess hardening:** lichess-bot config review (move overhead, TC ranges), disconnect/abort robustness, weird-position handling, multi-TC forfeit gauntlets.
- **Pondering (capstone):** UCI `go ponder`/`ponderhit`/miss handling + TimeBrain integration (ponder hits subsidize the clock). Worth ~+30-50 effective elo live (thinking on opponent time). Gate: ponder-enabled fastchess SPRT if the runner supports it cleanly; otherwise a documented soak (multi-hour ponder-on self-play with zero protocol violations/forfeits) + Lichess field telemetry as the verdict. Pulled forward from the original M9a because TimeBrain v1 is being built in the same plan — doing them together avoids reworking TM later.

Ship **v0.7.0**: same wrap discipline as every release.
